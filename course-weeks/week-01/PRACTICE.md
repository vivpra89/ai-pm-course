# Week 01 — Learning by practice

## Lab A — Environment (required)

Follow **[practice-env/README.md](../../practice-env/README.md)**:

1. Create `venv`, `pip install -r requirements.txt`.
2. Run `python scripts/build_dummy_db.py`.
3. Run `python labs/explore_metrics.py`.
4. Save terminal output (or screenshot) + **one sentence**: what the rollup measures.

## Lab B — Glossary (required)

Create `glossary.md` with **10 terms** from the master plan’s jargon list ([Phase 0 cheat sheet](../../TPM-PM-AI-Technical-Learning-Plan.md#phase-0--mental-model-and-jargon-week-1)). Define each **in your own words** in **≤ 2 lines**.

## Lab C — Context limits (required)

1. Open `data/long_briefing_stub.txt`. Imagine you must fit “the key decisions” into **≤ 150 words** for a VP.
2. Write that summary **yourself** (no LLM), then optionally compare with one LLM summary.
3. Write **5 bullets**: what you would **drop** first if the context budget shrinks again—tie drops to **business risk** (e.g. losing audit trail vs losing narrative).

## Lab D — Short essay (required)

In **≤ 250 words**, explain why **context limits** force **summarization**, **chunking**, or **RAG** for large corpora. Link to a **product** consequence (cost, wrong answers, stale policy).
